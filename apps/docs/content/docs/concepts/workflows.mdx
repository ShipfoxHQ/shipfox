A Shipfox **workflow** is a YAML file stored in your repository under `.shipfox/workflows/`. It defines the automation: what jobs to run, which steps to execute, and what triggers start a run. Workflows live in your repo, so they are versioned, diff-able, and reviewed like any other code.

A workflow has four moving parts. [Triggers](/concepts/triggers) decide when a run starts. [Jobs and steps](/concepts/jobs-steps) define the work. The `runner` label decides where jobs execute. `env` passes variables to shell steps.

## How it works

Shipfox scans `.shipfox/workflows/` in your connected repository for workflow definitions. Each file in that directory is one definition: **one file, one workflow**.

- **Any `.yml` or `.yaml` file under `.shipfox/workflows/` is picked up.** Files match by path prefix and extension, so nested paths such as `.shipfox/workflows/ci/build.yml` are scanned too.
- **The filename is not the workflow name.** The name comes from the top-level `name:` field inside the file. You can name the file anything you like.
- **Pushing a change updates the definition.** Shipfox detects new and edited workflow files automatically. There is no manual import.

## A complete workflow

```yaml
name: My workflow          # required; shown in the dashboard
runner: ubuntu-latest      # default runner label for all jobs
env:                       # env vars for run steps (not agent steps)
  NODE_ENV: production
triggers:                  # what starts a run
  on_push:
    source: github_acme    # your GitHub connection slug
    event: push
jobs:                      # required; at least one job
  build:
    steps:
      - run: npm test
```

Every field, its type, and its constraints are in the [Workflow Schema reference](/reference/workflow-schema).

**Env variables** exist at the workflow, job, and step level; the nearest scope wins, and they apply to `run` steps only. The full rules (naming, stringification, interpolation, size caps) live in [Environment variables](/reference/workflow-schema#environment-variables).
