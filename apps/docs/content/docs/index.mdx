**Your AI software factory.**

Shipfox turns your engineering work into pipelines that AI agents can run. You write each pipeline as YAML in your repo. Any integration event you define starts a run, such as a code push or an alert. Each agent reads context, runs your tools, then acts on the result. Every step streams to the dashboard as it runs.

## Anatomy of a pipeline

A pipeline starts with a trigger, then runs one or more jobs. Each job is an ordered list of steps, and a step is a shell command or an AI agent. Larger pipelines chain many jobs and run them in parallel.

```yaml
triggers:
  on_alert:                        # Define any trigger from your integrations
    source: sentry_prod
    event: issue.created
jobs:
  fix:
    steps:
      - run: npm install           # a shell step
      - harness: claude            # the Claude Agent SDK harness
        model: claude-opus-4-8
        prompt: Read the alert, reproduce it in the repo, and write a fix.
      - harness: pi                # the pi harness, with a custom model
        model: qwen3-coder
        prompt: /open-pr           # uses your existing skills and agent config
```

## Why Shipfox

- **Agents automate your process.** Any step can be an AI agent that reads your repo, runs tools, and acts on the result.
- **Watch every run.** Runs, jobs, and steps stream live to the dashboard. Agent steps show their messages, thinking, tool calls, tokens, and cost. See [Logging](/concepts/logging).
- **Works with your daily tools.** Start runs from your monitoring, ticketing, and chat tools, such as Sentry, Linear, and Slack, or from any webhook. See [Integrations](/integrations/overview).
- **Your harness, your models.** Pick your agent harness and any of the supported [model providers](/reference/model-providers).
- **Secure, ephemeral runners.** Jobs run on isolated [runners](/concepts/runner-provisioners) you can customize. Each one starts on demand and stops when idle.
- **Open source, MIT.** Self-host the whole platform. [Shipfox on GitHub](https://github.com/ShipfoxHQ/shipfox).
