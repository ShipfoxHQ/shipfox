A trigger has a `source` and an `event`. This page lists every source and its
events at a glance. For **integration** sources, the `source` you write is the
**connection slug** (for example `github_acme`), not the provider name. See
[Integrations](/integrations/overview). Setup and payloads live on each
integration page.

## Sources at a glance

| Source | `source` value | Events | Setup |
|---|---|---|---|
| Manual | `manual` (literal) | `fire` | [Events and triggers](/understand/events-and-triggers) |
| GitHub | connection slug, e.g. `github_acme` | `push` and other repository events | [GitHub](/integrations/github) |
| Sentry | connection slug, e.g. `sentry_acme` | `issue.created`, `issue.resolved`, `issue.assigned`, `issue.archived`, `issue.unresolved` | [Sentry](/integrations/sentry) |
| Custom webhook | connection slug you choose | `received` | [Custom webhook](/integrations/webhooks) |

| Slack | 🚧 Coming soon | Not available | [Slack](/integrations/slack) |
| GitLab | 🔜 Roadmap | Not available | Not available |

## manual

The built-in on-demand source. Use `manual` to start a workflow from the dashboard
or API without an external integration. At most one `manual` trigger per workflow.

```yaml
triggers:
  on_demand:
    source: manual
    event: fire
```

## Integration sources

For GitHub, Sentry, and custom webhooks, the `source` is the **connection slug**
created when you connect the integration. A single workspace can hold several
connections of the same provider, disambiguated by slug.

```yaml
triggers:
  on_push:
    source: github_acme
    event: push
```

See the [Integrations](/integrations/overview) section for setup, the full event
list per source, and payload details.

<Callout type="info">
  `event.ref` on a push is the **full Git ref** (for example `refs/heads/main`),
  exactly as the provider sends it, not the bare branch name.
</Callout>

## filter

The `filter` field accepts a CEL expression that narrows which events fire a
trigger. It is evaluated when the event arrives, before any run is created. In
scope: `event` (the payload) and `trigger` (`source` and `event`). A false
result skips the event. An expression error **fails closed**: the trigger does
not fire.

```yaml
triggers:
  on_main_push:
    source: github_acme
    event: push
    filter: event.ref == "refs/heads/main"
```

See [Expressions](/reference/expressions#trigger-filters) for the engine and
the exact scope.
