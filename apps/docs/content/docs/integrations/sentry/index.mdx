Sentry sends issue lifecycle events to Shipfox.

## Categories and related terms

**Category:** Observability.

**Related terms:** Errors, monitoring, crash reporting.

**Availability:** Available.

## Authentication

Shipfox connects through a Sentry App. Install the App in the target Sentry
organization and authorize its requested access.

## Integration connection identity

A Sentry integration connection slug defaults to `sentry_<organization>`, such
as `sentry_acme`. Use the workspace integration connection slug as a workflow
trigger `source`.

```yaml
triggers:
  on_issue:
    source: sentry_acme
    event: issue.created
```

For trigger field rules, see the [workflow schema reference](/reference/workflow-schema#trigger-fields).

| Capability | Availability |
