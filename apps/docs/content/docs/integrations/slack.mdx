<Callout type="info">
  **Coming soon.** The Slack integration is not available yet. This page
  describes where it is headed.
</Callout>

The Slack integration will connect a Slack workspace to Shipfox so conversations
and workflows can meet. Slack events will trigger workflow runs, and runs will
report results such as an agent summary, a failed check, or a finished triage back
to your channels.

Like every integration, it will follow the same model as [GitHub](/integrations/github)
and [Sentry](/integrations/sentry): connect once, get an **integration connection slug**
(e.g. `slack_acme`), and reference it as a trigger `source` with the events the
integration emits.

## Until it ships

Slack can already reach Shipfox through a [custom webhook](/integrations/webhooks):
create a webhook integration connection and point a Slack outgoing webhook or automation at its
ingest URL, then trigger on `received`. For the other direction, a `run`
step can post to a Slack incoming webhook today. Store the webhook URL as a
Shipfox secret and bind it through the run step's `env` map.
