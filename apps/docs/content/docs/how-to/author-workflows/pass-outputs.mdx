---
title: "Pass Outputs Between Shipfox Steps and Jobs"
sidebarTitle: "Pass Outputs"
description: "Declare typed step outputs, publish selected values from a job, and read them in later work."
---

Use outputs for small values that later work needs. Jobs do not share files or
process environments.

## Produce a step output

Declare the output and append its value to `$SHIPFOX_OUTPUT`:

```yaml
jobs:
  package:
    steps:
      - key: read_version
        run: echo "version=$(git describe --tags --always)" >> "$SHIPFOX_OUTPUT"
        outputs:
          version: string
```

Later steps in the same job can read
`${{ steps.read_version.outputs.version }}`.

## Publish it to another job

Expose the step value from the job. Add a dependency to the next job. Bind the
value to an environment variable before using it in a command:

```yaml
jobs:
  package:
    outputs:
      version: ${{ steps.read_version.outputs.version }}
    steps:
      - key: read_version
        run: echo "version=$(git describe --tags --always)" >> "$SHIPFOX_OUTPUT"
        outputs:
          version: string

  publish:
    needs: package
    steps:
      - env:
          VERSION: "${{ jobs.package.outputs.version }}"
        run: printf 'Publishing version %s\n' "$VERSION"
```

Start the workflow and inspect both jobs in the run detail. The `package` job
should expose `version`. The `publish` job should print the same value. For JSON
outputs and limits, see [Step outputs](/reference/workflow-schema#step-outputs).
