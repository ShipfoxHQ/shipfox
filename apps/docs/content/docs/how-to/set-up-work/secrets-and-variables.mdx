title: "Add Secrets and Variables to Shipfox Workflows"
sidebarTitle: "Secrets & Variables"
description: "Store workspace configuration safely and bind variables or secrets into workflow steps."
---

Use a variable for readable configuration and a secret for a credential. Both
are available to projects in the workspace.

## Add a variable

1. Open **Settings → Variables** in the workspace.
2. Select **Create variable**.
3. Enter an uppercase key such as `DEPLOY_TARGET` and its value.
4. Save the variable.

Read it from a template:

```yaml
jobs:
  deploy:
    steps:
      - env:
          TARGET: "${{ vars.DEPLOY_TARGET }}"
        run: ./deploy.sh "$TARGET"
```

## Add a secret

1. Open **Settings → Secrets**.
2. Select **Create secret**.
3. Enter an uppercase key such as `NPM_TOKEN` and the secret value.
4. Save it. Shipfox does not show the value again.

Bind the secret only to a run step:

```yaml
jobs:
  publish:
    steps:
      - env:
          NPM_TOKEN: "${{ secrets.NPM_TOKEN }}"
        run: npm publish
```

Commit and push the workflow, then start a run. A missing variable stops run
creation. A missing secret fails the step before the command runs. Storage,
masking, and scope rules live in [Secrets and variables](/reference/secrets-variables).
